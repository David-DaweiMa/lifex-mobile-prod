# Coly V2 PRD（用户故事 / 验收 / 数据字典 / 提醒策略 / KPI）

> Guiding Principle（Lifex Manifesto）
> 我们的核心价值是利用 AI 的能力深入理解每个 business 和每个用户，并据此做最合适的匹配。所有可以合法获取的数据都用于帮助 AI 持续加深理解；随着时间与数据增长，这份理解会不断进化。唯有基于这种理解，我们才能提供让客户满意的一切服务（chat、推荐、AI 助手）以及后续的优惠券、订座、订餐、服务预定、票务购买等下游交易。

## 1. 背景与目标
- 通过“少而准”的省钱/省时服务提升订阅转化与留存，隐私优先、成本可控。
- V2 重点：到期提醒、历史价图与替代推荐、First Table 名额提醒、共享清单（可选）。

## 2. 范围（In / Out）
- In：
  - 到期提醒（WOF/rego/券/卡/会员/订阅）
  - 价格历史（Top SKU/服务项）与替代推荐
  - First Table 名额提醒（Top 餐厅）
  - 周报升级与频控
- Out：
  - 银行/邮箱读取；UGC抓取；全量SKU实时价

### 扩展：燃油价格（Fuel）轻量版（可选纳入V2）
- 城市本周最低 + 附近更便宜 ×3（指示性价，日/隔日更）
- 我的油站关注 + 阈值提醒；路线“顺路更省”提示

#### 第三方平台说明
- Gassy（`https://www.gassy.co.nz/`）与 Gaspy 不同；采用“合作/深链”并保留来源标注，不抓取其内容。

### 扩展：EV 充电（轻量版）
- 用户故事：
  - 作为车主，我希望看到附近≥50kW 的快充并按距离/功率排序。
  - 作为出行用户，我希望路线沿途推荐合适停靠点，减少等待与绕路。
- 验收要点：
  - 支持按接口/功率筛选；提供外部深链；标注来源/更新时间。
  - 若无实时可用性，仅展示“指示性”并提示以现场/官方为准。


## 3. 用户故事与验收标准（示例）
- US-001：作为用户，我可以手动添加 WOF 到期（日期/周期），到期前 48h 收到提醒。
  - AC：
    - 支持类型=wof，必填 due_date；保存成功显示在 Reminders。
    - 默认 48h 前提醒；可关闭或调整。
- US-002：作为用户，我关注的 SKU 价格下降 ≥10% 时收到提醒。
  - AC：
    - user_watchlist 记录阈值；价格变动批处理命中生成通知。
    - 通知含旧价/新价/门店/有效期；点击进入比价页。
- US-003：作为用户，我对心愿餐厅的 First Table 心愿时段出现席位时收到提醒。
  - AC：
    - first_table_slots 日更2次；命中 watchlist（business+timeslot）即生成通知。
- US-004：作为用户，我的“每周省钱简报”合并显示本周命中：超市、餐饮、理发、周末计划。
  - AC：
    - 限定 5–8 条卡片；支持“稍后提醒/不再推荐”。

## 4. 数据字典（新增/关键）
- reminders
  - id, user_id, type, title, due_date, period(enum: none/weekly/monthly/yearly), notes, is_active, created_at, updated_at
- user_watchlist
  - id, user_id, watch_type(enum: business/sku/service), ref_id, threshold_percent, threshold_amount, created_at
- supermarket_prices（补充）
  - id, store_id, sku_id, price, promo_text, valid_from, valid_to, scraped_at
- deals / first_table_slots（同 DB 提案）
 - fuel_stations / fuel_prices（指示性价格）

## 5. 通知与频控策略
- 默认：每周简报（周五上午），实时仅阈值/到期命中。
- 去打扰：每用户每周 ≤2 次推送；低命中源自动降频；支持静默仅App内。
- 卡片内容规范：标题（含节省数字/时段）、描述（简短）、行动（预订/导航/比价/加入清单）。
  - Fuel 卡片加“更新时间/来源”标注；提示“以到店为准”。

## 6. 数据刷新与计算
- 批处理：
  - 周更：超市促销/价目、官网优惠、菜单/价目；
  - 日更：First Table；热点优惠。
- 命中计算：
  - 价格降幅 = (old_price - new_price) / old_price；
  - 阈值比较使用用户设置（或默认10% / $5）。
- 历史价图：
  - 仅 Top SKU/服务项；最近 3/6/12 个月（采样点每周）。

## 7. 隐私与合规
- 仅公开源；遵守 robots；速率限制；不抓UGC。
- 端侧画像；云端仅聚合与命中日志；一键清除。

## 8. KPI 与度量
- 打开率≥35%，点击率≥12%，降价提醒点击≥18%。
- 人均月节省≥订阅价×4；周报卡片“有用”反馈≥60%。
- 抓取失败率<3%；数据覆盖≥Top 店铺/SKU 80%。

## 9. 里程碑（6周）
- W1：数据字典与UI稿；提醒与阈值方案评审
- W2：到期提醒数据结构与UI；价格历史采样逻辑
- W3：降价命中与通知；First Table 映射与日更
- W4：周报模块化卡片；文案A/B; 频控实现（策略文档）
- W5：替代推荐规则（同城同类性价比）；质量监控指标
- W6：灰度试运行与指标验证；总结与优化清单
